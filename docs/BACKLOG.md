# Product Backlog

Managed by the Product Manager agent. Items are prioritized by user value and pulled into delivery cycles via `docs/DELIVERY_PLAN.md`.

## Backlog Protocol

1. New requirements discovered mid-cycle are added here, not to the current cycle.
2. Product Manager owns prioritization and grooming.
3. Orchestrator pulls items from the backlog into `docs/DELIVERY_PLAN.md` at the start of each cycle.
4. Only blocking items may be promoted mid-cycle (requires Product Manager + Orchestrator approval).
5. Completed items remain in the backlog with `done` status for traceability.

## Priority Legend

- **P0** (Must-have): Blocking or critical for current delivery.
- **P1** (Should-have): Important for user value but not blocking.
- **P2** (Nice-to-have): Low-impact or deferrable.

## Status Legend

- `open` — Not yet planned for a cycle.
- `planned` — Pulled into a delivery cycle.
- `in-progress` — Currently being worked on.
- `done` — Completed and validated.
- `deferred` — Intentionally postponed with rationale.
